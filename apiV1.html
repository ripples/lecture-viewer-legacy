Lecture Viewer API

LectureViewer API will be utilized by the React front end and will allow for CRUD operations by being the middle man for the database.

This API is almost complete. There may still be some minor errors. You shouldn’t expect any more drastic changes.

Authentication

Almost every API call defined in this document will require a user to be logegd in. To verify that you are logged in, you will pass a token to the backend with every API request you make. You can do this by setting the “Authorization” header in your request to pass the token that is returned by login.

    Authorization: eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9

API Responses

These are examples of the responses that one can expect to recieve from the API

A Successful Response

This shows a response example of what a correctly used api with no errors would look like

    
{
    "status" : "success",
    "data" :
        {
            "first_name":"Jane",
            "last_name":"Doe",
            "email":"jdoe@umass.edu",
            "role":"student",
            "courses":[],
            "user_id":"55366107d76f149f57d3ee56"
        }
}

A Failed Response

This shows a response example of what an improperly formatted api call looks like

    
{
    "status" : "fail",
    "data" : {"message":"Incorrect parameters"}
}

An Error Response

This shows a response example of what an internal error would return

    
{
    "status" : "error",
    "data" : { "message" : "There was a database connection error" }
}
